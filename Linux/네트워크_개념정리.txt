ARP
2계층(데이터 링크 계층)**과 3계층(네트워크 계층) 사이의 프로토콜로 간주됩니다. ARP는 네트워크 계층의 IP 주소를 데이터 링크 계층의 MAC 주소로 변환하는 역할을 하기 때문입니다.

IP 주소를 MAC 주소로 변환해 실제 데이터 통신이 가능하도록 함
브로드캐스트 요청(모두에게 알림) → 응답 수신 → MAC 주소 획득
네트워크 상의 장치들이 서로 데이터를 주고받기 위해 필수적인 과정
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
ICMP

3계층(네트워크 계층)**에 속합니다. ICMP는 주로 네트워크 계층에서 네트워크 상태를 진단하고 오류를 보고하는 데 사용됩니다.

네트워크 상태 확인:
A 컴퓨터가 B 컴퓨터에 "Ping" 메시지를 보냅니다.
B 컴퓨터가 "Pong" 메시지로 응답합니다.
A 컴퓨터는 B 컴퓨터가 정상적으로 네트워크에 연결되어 있음을 확인합니다.
문제 발생 알림:
A 컴퓨터가 B 컴퓨터에 데이터를 보내려고 합니다.
중간의 네트워크 장비(예: 라우터)에서 문제가 발생하여 데이터를 전달할 수 없습니다.
네트워크 장비는 ICMP 메시지를 통해 A 컴퓨터에게 "데이터를 보낼 수 없어요!"라고 

즉 ICMP는 네트워크에서 장비들이 서로 통신 상태를 점검하고 문제가 생겼을 때 신속하게 
알림을 보내어 문제를 해결할 수 있도록 도와주는 중요한 역할을 합니다.
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
DNS

응용 계층(7계층): 이 계층에서는 네트워크 서비스를 사용자와 애플리케이션에 제공하는 역할을 합니다. 웹 브라우징, 이메일 전송, 파일 전송 등의 서비스가 이 계층에서 이루어집니다.
DNS의 역할: DNS는 사람이 기억하기 쉬운 도메인 이름(예: www.example.com)을 컴퓨터가 이해할 수 있는 IP 주소(예: 192.0.2.1)로 변환하는 서비스입니다. 이 과정은 사용자가 웹사이트에 접근하거나 이메일을 보낼 때 필수적입니다.

목적: 사람이 이해할 수 있는 도메인 이름을 컴퓨터가 이해할 수 있는 IP 주소로 변환.
과정: 도메인 이름 입력 → DNS 서버에서 IP 주소 찾기 → IP 주소 반환 → 웹사이트 접속.
중요성: 인터넷 사용을 쉽고 편리하게 만들어줌.
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
DHCP

응용 계층(7계층): 이 계층에서는 네트워크 서비스를 사용자와 애플리케이션에 제공하는 역할을 합니다. DHCP는 네트워크 상에서 자동으로 IP 주소를 할당하고, 네트워크 설정을 제공하는 서비스를 수행합니다.
DHCP의 역할: DHCP는 네트워크에 연결된 장치들에게 IP 주소, 서브넷 마스크, 게이트웨이, DNS 서버 주소 등을 자동으로 할당하여 네트워크 설정을 간편하게 합니다. 이를 통해 네트워크 관리자는 수동으로 설정할 필요 없이 장치들이 자동으로 네트워크에 접속할 수 있도록 할 수 있습니다.

목적: 네트워크 장치들이 자동으로 IP 주소 및 네트워크 설정을 받을 수 있도록 함.
과정: 장치 연결 → IP 주소 요청 → DHCP 서버 응답 → IP 주소 할당.
중요성: 네트워크 설정을 자동화하여 사용자가 손쉽게 네트워크에 연결할 수 있도록 도와줌.

즉 DHCP는 네트워크 상에서 장치들이 IP 주소와 기타 필요한 설정을 자동으로 받을 수 있게 
해주는 중요한 시스템입니다.
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
HTTP

응용 계층(7계층): 이 계층에서는 사용자와 애플리케이션이 네트워크를 통해 다양한 서비스를 이용할 수 있도록 합니다. 웹 브라우징, 이메일 전송, 파일 전송 등의 서비스가 이 계층에서 이루어집니다.
HTTP의 역할: HTTP는 웹 브라우저와 웹 서버가 웹페이지와 같은 리소스를 주고받는 데 사용하는 프로토콜입니다. 사용자가 웹 브라우저에 URL을 입력하면, HTTP를 통해 웹 서버에 요청을 보내고, 웹 서버는 요청된 리소스를 HTTP를 통해 응답으로 돌려줍니다.


목적: 웹 브라우저와 웹 서버 간에 웹페이지와 데이터를 주고받기 위한 규칙.
과정: 브라우저가 요청 → 서버가 응답 → 브라우저가 웹페이지 표시.
중요성: 인터넷을 통해 웹페이지를 쉽게 접근하고 볼 수 있게 해줌.

즉 HTTP는 우리가 웹사이트를 방문하고 웹페이지를 보는 과정을 가능하게 해주는 
중요한 프로토콜입니다.
